# 亚马逊上传表格通用自检资料

用途：给自动填表流程或人工复核使用。  
适用范围：Amazon 上传添加商品模板，尤其是 Haul / 低价商城新品表格。  
目标：减少因字段漏填、单位不一致、品牌路线不一致、变体关系错误导致的上传失败。

## 1. 填表前确认

每个产品开始填写前，先确认这些基础信息：

| 检查项 | 要确认的内容 |
|---|---|
| 商品类目 | product_type 和 item_type_keyword 是否匹配实际商品 |
| 是否无品牌 | 走无品牌时 Brand / Manufacturer / 标题 / 描述 / 卖点都不能出现供应商名 |
| 是否多款式 | 多颜色、多尺寸、多规格是独立 SKU 还是父子变体 |
| SKU 规则 | SKU 是否唯一，是否包含款式差异词 |
| 尺寸单位 | 使用 inches 还是 centimeters，整份表格必须统一 |
| 重量单位 | 商品重量、包装重量使用 Grams 或 Pounds，避免混用 |
| 售价 | 普通价格和低价商城 / Haul 价格是否都填对 |
| 图片 | 是否已有可公开访问的图片 URL |
| 合规 | 是否涉及电池、危险品、农药设备、FCC、加州 65 等 |

## 2. 无品牌路线规则

走无品牌路线时：

| 字段 | 推荐填写 |
|---|---|
| Brand Name | `Generic` |
| Manufacturer | `Generic` |

同时检查：

```text
标题中不能出现供应商公司名
描述中不能出现供应商公司名
卖点中不能出现供应商公司名
关键词中不能出现供应商公司名
不要把采购单上的公司名当品牌填入
```

常见错误：

```text
Brand 填了供应商名
Manufacturer 填了供应商公司全称
标题开头带了品牌名，但实际准备走无品牌
有效值页里只有某个品牌，不代表必须填那个品牌，除非平台/账号强制要求
```

## 3. SKU 和款式一致性

SKU 应该能反映款式差异，但不要建立不需要的变体。

多款式独立 SKU 的推荐规则：

```text
基础品名-颜色
基础品名-尺寸
基础品名-规格
基础品名-颜色-尺寸
```

自检：

```text
SKU 是否唯一
SKU 中的颜色是否与 Color 字段一致
SKU 中的尺寸是否与 Size 字段一致
SKU 中不要使用中文、空格或特殊符号
如果是独立 SKU，不要填写父 SKU 或变体主题
```

## 4. 变体字段规则

如果商品不需要做父子变体，以下字段必须留空：

```text
parentage_level
child_parent_sku_relationship.parent_sku
variation_theme
```

不要填：

```text
Parent
Child
COLOR
SIZE
COLOR/SIZE
```

除非明确要创建父子变体。

当前系统支持三种无品牌路线：

```text
Haul Generic
单链接/独立子体：只生成可售 SKU，不填写 parentage、parent_sku、variation_theme。

Haul Generic Variation
多链接变体：生成 1 个 Parent SKU 和多个 Child SKU。Parent 填 parentage_level=Parent 和 variation_theme；Child 填 parentage_level=Child、parent_sku 和相同 variation_theme。

Haul Generic Set Bundle
单链接套装售卖：只生成可售 Child SKU，不建立 Parent/Child；set_count、number_of_items、包装内容按套装数量填写。
```

如果要创建变体，需要另外确认：

```text
父 SKU 是否存在
子 SKU 是否都填了 parent_sku
variation_theme 是否与实际差异字段一致
父项是否不填写价格、库存、尺寸、颜色等子项专属字段
每个子项是否都有完整标题、卖点、图片和报价字段
```

## 5. 标题、描述、卖点

标题原则：

```text
英文填写
不要全大写
不要堆砌关键词
不要直接照抄竞品标题
无品牌路线不要放品牌词
颜色、尺寸、数量等款式信息要与字段一致
```

描述原则：

```text
说明产品是什么、用途、材质、尺寸、适用场景
不要写无法证明的夸大词，如 best、top、guaranteed、100% safe
不要写平台禁止内容、售后承诺、物流承诺
不要出现供应商名、竞品品牌、ASIN
```

卖点原则：

```text
每条只讲一个核心卖点
优先覆盖用途、材质、尺寸重量、适用场景、包装数量
不要写促销语
不要写和字段冲突的信息
```

## 6. 单位规则

尺寸字段要统一。

如果平台要求 inches：

```text
所有尺寸单位字段填 Inches
文案中也使用 inches
不要残留 cm、Centimeters、厘米
包装尺寸和商品尺寸也要统一换算
```

常用换算：

```text
1 inch = 2.54 cm
inch = cm / 2.54
```

重量字段：

```text
如果重量来自克，单位填 Grams
如果换算磅，1 lb = 453.592 g
商品重量和包装重量要分别检查
不要把克数填到 Pounds 单位下
```

自动校验建议：

```text
扫描商品行是否包含 cm、Centimeters、厘米
扫描尺寸单位列是否全部为 Inches 或目标单位
扫描重量单位列是否与数值来源一致
```

## 7. 价格字段

低价商城 / Haul 表格通常有专用价格字段。

自检：

```text
list_price 是否填写
低价商城 / Haul / BZR price 是否填写
价格是否低于该类目的 Haul 价格阈值
最低价、最高价字段如果填写，是否不会和售价冲突
价格字段不要带 $ 符号，只填数字
```

## 8. 图片字段

图片 URL 未准备好时：

```text
可以先留空
不要填本地路径
不要填竞品图片 URL
不要填 1688 商品页截图地址
```

图片 URL 准备好后：

```text
必须是公开可访问 URL
主图应是白底、无水印、无文字、只展示所售商品
不同颜色 SKU 应尽量使用对应颜色主图
不要使用不随商品配送的道具造成误导
```

## 9. 合规字段

合规字段不要机械套用，要先判断产品属性。

### 电池

非电动、非含电池产品：

```text
batteries_required = No
batteries_included = No
不要填写电压、瓦数、电池类型、锂电池参数
```

含电池产品：

```text
必须确认电池类型、数量、是否内置、锂电池参数、包装方式
不要随便填 No
```

### 危险品

普通非危险品：

```text
supplier_declared_dg_hz_regulation = Not Applicable
不要填写 GHS 分类、Hazmat 编号、SDS URL
```

如果是液体、化学品、喷雾、带电池等，要重新判断。

### 农药 / 农药设备

如果不是农药或农药设备：

```text
pesticide_marking.registration_status = This product is not a pesticide or pesticide device, as defined under the U.S. Federal Insecticide, Fungicide, and Rodenticide Act.
```

注意：

```text
某些模板会要求 pesticide_marking.marking_type 同时有值
否则可能报 99022：marking_type does not have enough values
如遇到该错误，可按模板有效值补 marking_type，例如 EPA Registration Number
不要随便填写农药认证编号，除非产品确实有对应编号
```

### FCC

非电子、不会发射射频的产品：

```text
不要填写 FCC ID
不要填写 SDoC 联系人
```

电子产品、无线产品、蓝牙/Wi-Fi 产品需要单独确认 FCC 字段。

## 10. 字段联动检查

很多报错来自“只填了一个字段，没填配套字段”。

常见联动：

| 已填写字段 | 可能需要同步填写 |
|---|---|
| 尺寸数值 | 尺寸单位 |
| 重量数值 | 重量单位 |
| 价格 | 币种或平台专用价格字段 |
| 变体主题 | 父子关系字段 |
| 电池 required/included = Yes | 电池类型、数量、包装、重量 |
| GHS / Hazmat | SDS URL、危险品编号 |
| pesticide registration_status | pesticide marking_type |
| 包装级别不是 Unit | 包含 SKU 数量和 SKU 编码 |

## 11. 上传前自动校验清单

逐行检查：

```text
SKU 不为空且唯一
product_type 不为空
record_action 是否正确
标题为英文且不含不该出现的品牌词
Brand 和 Manufacturer 与品牌路线一致
Color / Size / SKU 款式词一致
独立 SKU 的 parentage / parent_sku / variation_theme 为空
价格字段为数字，不带 $ 符号
商品尺寸和包装尺寸都有单位
商品重量和包装重量都有单位
文案单位和字段单位一致
电池字段与产品实际属性一致
危险品字段与产品实际属性一致
农药字段没有缺少配套 marking_type
图片 URL 没有误填本地路径或竞品链接
没有残留示例行中文提示
没有把供应商公司名误填进标题、品牌、制造商、描述、卖点
```

## 12. 错误报告复盘方法

拿到 processing summary 后：

```text
先看 Feed Processing Summary
记录错误代码、错误字段、受影响 SKU
优先处理所有 SKU 共同出现的同一错误
对照“受影响字段（单元格）”定位源文件字段
注意 processing summary 可能比源文件多出状态列，列号可能右移
不要直接修改 processing summary 当作上传文件，应该修源文件
修复后重新保存源文件并再次上传
```

常见错误处理：

```text
99022: 某个复合属性缺少必填子字段，检查同组字段是否只填了一部分
单位错误: 检查有效值页，单位大小写和拼写要用模板允许值
品牌错误: 检查 Brand 是否允许 Generic，标题是否与品牌路线冲突
变体错误: 检查 Parent/Child、parent_sku、variation_theme 是否配套
```
